# 🔐 Security Policy

*Found a bug in the framework itself? Here's how to report it responsibly.*

</div>

---

## ⚠️ Important

> **Do NOT open a public GitHub issue for security vulnerabilities.**
>
> Public disclosure before a fix is in place puts all users at risk.

---

## 📬 How to Report

Send an email to **security@maeitsec.github.io** with the following:

| Field | Details |
|-------|---------|
| **Description** | What is the vulnerability? |
| **Steps to Reproduce** | How can it be triggered? |
| **Potential Impact** | What could an attacker do with it? |
| **Suggested Fix** | Optional, but appreciated |

We'll handle your report with full confidentiality.

---

## ⏱️ What to Expect

```
Day 0    You send the report
  │
  ▼
Day 1-2  Acknowledgment from maintainers
  │
  ▼
Day 3-7  Status update + impact assessment
  │
  ▼
  ?      Fix developed and tested
  │
  ▼
  ?      Coordinated disclosure (with your input)
```

We will work with you to understand and resolve the issue **before any public disclosure**.

---

## 🎯 Scope

### ✅ In scope

- Python scripts included in this repository
- `SKILL.md` configurations and logic
- Any infrastructure directly operated by this project

### ❌ Out of scope

- Vulnerabilities found **in third-party targets** using these tools
  *(report those directly to the target's security team)*
- Issues with third-party dependencies (Go tools, system utilities)
- Bugs that don't have a security impact

---

## 🛡️ Responsible Use

These tools are designed for **authorized security testing**. If you encounter situations where the tools behave unexpectedly or could cause unintended harm — even outside a traditional vulnerability context — please report it so we can add appropriate safeguards.
